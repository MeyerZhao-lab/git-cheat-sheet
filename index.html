Git 速查表 备忘单

安装

Git 官网：git-scm.com

配置工具

对你的commit操作设置关联的用户名

$ git config --global user.name "[name]"

对你的commit操作设置关联的邮箱地址

$ git config --global user.email "[address]"

启用有帮助的彩色命令输出

$ git config --global color.ui auto

Create && Remote

当着手于一个新的仓库时，你只需创建一次。要么在本地创建，然后推送到 GitHub；要么通过 clone 一个现有仓库。

cd ~/my_peoject_directory
git init

在使用过 git init 命令后，使用以下命令将本地仓库与一个 GitHub 上的空仓库连接起来：

git remote add origin [url]

将现有目录转换为一个 Git 仓库

git clone [url]

生成SSH秘钥

你可以按如下命令来生成 sshkey:

ssh-keygen -t ed25519 -C "your_email@xxx.com"

按照提示完成三次回车，即可生成 ssh key。通过查看 ~/.ssh/id_ed25519.pub 文件内容，获取到你的 public key

cat ~/.ssh/id_ed25519.pub

复制生成后的 ssh key，通过仓库主页 「管理」->「部署公钥管理」->「添加部署公钥」 ，添加生成的 public key 添加到仓库中。

添加后，在终端（Terminal）中输入 

ssh -T git@github.com
ssh -T git@github2.com

ssh -T git@gitee.com

查看修改

查看修改历史

git log

列出文件的历史版本

git log --fllow [file]

搜索log

git log --grep=<pattern>

查看未暂存代码修改

git diff

查看已暂存代码修改

git diff --staged

比较两个分支之间的差异

git diff [first-branch]...[second-branch]

查看当前分支最新的log

git show
# or
git show [commit]

分支管理 branch

创建一个新的分支(基于当前分支)

git branch [branch-name]

切换到新创建分支

git checkout [branch-name]

创建并切换到新分支

git checkout -b [branch-name]

创建一个新的分支(基于其他分支)

git checkout -b [branch-name] [branch-other]

返回上一次分支

git checkout -

删除分支

git branch -d [branch-name]

Merge FeatureA into Master

git checkout Master
git merge FeatureA

同步更改

同步仓库代码

git fetch

查看修改的代码

# 显示不在 featureA, 在 origin/featureA的log
git log featureA..origin/featureA

将代码合并进本地分支

git merge origin/featureA

提交自己的本地修改

git commit -am 'small tweak'

将所有本地分支提交上传到 GitHub

git push

使用来自 GitHub 的对应远端分支的所有新提交更新你当前的本地工作分支。

# git pull 是 git fetch 和 git merge 的结合
git pull

Cherry-pick

如果你希望将提交 e43a6 拉取到 master 分支，你可以运行：

git checkout master
git cherry-pick e43a6

Reset

当需需要代码回滚到前一个版本

git reset --hard HEAD^
git reset --hard commid_id # 回滚代码到制定版本

Checkout

查看commit 的代码(不用 rest)

git checkout commit-id

从 commit-id 检出分支

git checkout -b branch-name commit-id

储藏工作

当前工作开发到一半 需要切换分支做其他工作，需要暂时保存当前分支的代码

git stash
git stash list # 查看储藏列表
git stash apply # 应用最新的储藏
git stash apply # 引用指定储藏
git stash pop # 应用最新储藏，然后直接扔掉 删除
git stash drop # 删除储藏
git stash --all

从储藏检出分支

git stash branch textchange

Clean

删除未追踪的文件 清空工作目录

git clean -d -n
git clean -f -d

从储藏检出分支

git stash branch textchange

帮助

查看文档帮助

git <COMMAND> --help

e.g.

git config --help

你也可以使用 git help config， 在跟一个选项 -w 打开网页版

git help config -w

.gitignore 文件

有时一些文件最好不要用 Git 跟踪。这通常在名为 .gitignore 的特殊文件中完成。你可以在 github.com/github/gitignore 找到有用的 .gitignore 文件模板。

术语表

  • git: 一个开源的分布式版本控制系统
  • GitHub: 一个托管和协作管理 Git 仓库的平台
  • commit 提交: 一个 Git 对象，是你整个仓库的快照的哈希值
  • branch 分支: 一个轻型可移动的 commit 指针
  • clone: 一个仓库的本地版本，包含所有提交和分支
  • remote 远端: 一个 GitHub 上的公共仓库，所有小组成员通过它来交换修改
  • fork: 一个属于另一用户的 GitHub 上的仓库的副本
  • pull request 拉取请求: 一处用于比较和讨论分支上引入的差异，且具有评审、评论、集成测试等功能的地方
  • HEAD: 代表你当前的工作目录。使用git checkout 可移动 HEAD 指针到不同的分支、标记(tags)或提交
